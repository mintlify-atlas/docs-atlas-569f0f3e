# Introduction to tsim

tsim is a quantum circuit simulator that enables fast sampling from Clifford+T circuits with Pauli noise. It feels just like [Stim](https://github.com/quantumlib/Stim), but with support for non-Clifford gates like T, T_DAG, and arbitrary rotation gates.

## What is tsim?

tsim is a quantum circuit sampler designed for efficient sampling of circuits that combine Clifford gates with non-Clifford operations like T gates. Its underlying algorithm uses stabilizer rank decomposition together with ZX-calculus simplification rules, based on [Sutcliffe and Kissinger (2025)](https://arxiv.org/abs/2403.06777).

Just like Stim, tsim compiles circuits into measurement or detector samplers. These samplers manage a contiguous data structure that allows for efficient sampling on CPU or GPU.

## Key Differences from Stim

While tsim maintains API compatibility with Stim, there are important differences:

<Note>
**Detector Sampling Behavior**: tsim simulates actual detector samples, whereas Stim only reports detection flips (detection samples XORed with a noiseless reference sample).

For example, a circuit with `X 0; M 0; DETECTOR rec[-1]` will report `True` values in tsim but `False` values in Stim.
</Note>

<Info>
**Performance Trade-offs**: tsim can simulate hundreds of qubits, as long as the circuit does not have too many non-Clifford instructions. The stabilizer rank grows with the number of T gates, so circuits with many T gates may become computationally expensive.
</Info>

## Supported Operations

tsim supports all [Stim instructions](https://github.com/quantumlib/Stim/wiki/Stim-v1.9-Gate-Reference), including:

- All Clifford gates (H, S, CNOT, CZ, SQRT_X, etc.)
- All Pauli gates (X, Y, Z)
- All Stim noise channels (DEPOLARIZE1, DEPOLARIZE2, PAULI_CHANNEL_1, X_ERROR, etc.)
- Measurement operations (M, MR, MX, MY, MZ)
- Detector and observable annotations

**Additionally**, tsim supports these non-Clifford instructions:

- **T and T_DAG**: π/4 phase rotation gates
- **R_X, R_Y, R_Z**: Rotation gates around the X, Y, and Z axes
- **U3**: General single-qubit unitary with three parameters

## How It Works

tsim uses stabilizer rank decomposition based on the ZX calculus and is built on top of [pyzx](https://github.com/Quantomatic/pyzx). The workflow is:

1. Circuits are converted into ZX diagrams
2. Noise channels are injected as parametrized Pauli vertices
3. ZX-calculus simplification rules reduce the diagram complexity
4. The diagram is compiled into contiguous JAX arrays for efficient sampling

This approach enables fast sampling on both CPU and GPU, following the methodology described in the [Sutcliffe and Kissinger paper](https://arxiv.org/abs/2403.06777).
