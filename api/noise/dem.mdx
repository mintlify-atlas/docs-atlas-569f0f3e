## Overview

The `tsim.noise.dem` module provides functionality to generate detector error models (DEMs) from Stim circuits. It extends Stim's built-in DEM generation with support for non-deterministic observables, which is critical for certain quantum error correction protocols.

## get_detector_error_model

```python
def get_detector_error_model(
    stim_circuit: stim.Circuit,
    *,
    allow_non_deterministic_observables: bool = True,
    decompose_errors: bool = False,
    flatten_loops: bool = False,
    allow_gauge_detectors: bool = False,
    approximate_disjoint_errors: bool = False,
    ignore_decomposition_failures: bool = False,
    block_decomposition_from_introducing_remnant_edges: bool = False
) -> stim.DetectorErrorModel
```

Return a `stim.DetectorErrorModel` describing the error processes in the circuit.

Unlike the `stim.Circuit.detector_error_model()` method, this function allows for non-deterministic observables when `allow_non_deterministic_observables` is set to true. This is achieved by converting logical observables into detectors, calling `stim.Circuit.detector_error_model(allow_gauge_detectors=True)`, and then reconverting the detectors back into observables in the detector error model.

<Warning>
  If the circuit has distance one (i.e., if there are errors that only flip logical observables), this method will return an incorrect detector error model.
</Warning>

### Parameters

<ParamField path="stim_circuit" type="stim.Circuit" required>
  The Stim circuit to compute the detector error model for.
</ParamField>

<ParamField path="allow_non_deterministic_observables" type="bool" default="True">
  When set to true, the detector error model allows for non-deterministic observables. This is achieved by converting logical observables into detectors, calling `stim.Circuit.detector_error_model(allow_gauge_detectors=True)`, and then reconverting the detectors back into observables.
  
  <Note>
    This is the key difference from Stim's built-in `detector_error_model()` method and is particularly useful for gauge-fixing in surface codes and other topological codes.
  </Note>
</ParamField>

<ParamField path="decompose_errors" type="bool" default="False">
  When set to true, the error analysis attempts to decompose composite error mechanisms (such as depolarization errors) into simpler errors, suggesting this decomposition via `stim.target_separator()` between components.
  
  For example, in an XZ surface code, single qubit depolarization has a Y error term which can be decomposed into simpler X and Z error terms.
  
  <Warning>
    Decomposition fails (causing this method to throw) if it's not possible to decompose large errors into simple errors that affect at most two detectors. When `allow_non_deterministic_observables` is true, decomposition is not supported.
  </Warning>
</ParamField>

<ParamField path="flatten_loops" type="bool" default="False">
  When set to true, the output will not contain any `repeat` blocks. When set to false, the error analysis watches for loops in the circuit reaching a periodic steady state with respect to the detectors being introduced, the error mechanisms that affect them, and the locations of the logical observables.
  
  When a steady state is identified, it outputs a repeat block. This is massively more efficient than flattening for circuits that contain loops, but creates a more complex output.
  
  <Note>
    This parameter is irrelevant unless `allow_non_deterministic_observables=False`.
  </Note>
</ParamField>

<ParamField path="allow_gauge_detectors" type="bool" default="False">
  When set to false, the error analysis verifies that detectors in the circuit are actually deterministic under noiseless execution of the circuit. When set to true, these detectors are instead considered to be part of degrees of freedom that can be removed from the error model.
  
  For example, if detectors D1 and D3 both anti-commute with a reset, then the error model has a gauge `error(0.5) D1 D3`. When gauges are identified, one of the involved detectors is removed from the system using Gaussian elimination.
  
  <Note>
    Logical observables are still verified to be deterministic, even if this option is set.
  </Note>
</ParamField>

<ParamField path="approximate_disjoint_errors" type="bool | float" default="False">
  When set to false, composite error mechanisms with disjoint components (such as `PAULI_CHANNEL_1(0.1, 0.2, 0.0)`) can cause the error analysis to throw exceptions (because detector error models can only contain independent error mechanisms).
  
  When set to true, the probabilities of the disjoint cases are instead assumed to be independent probabilities. For example, a `PAULI_CHANNEL_1(0.1, 0.2, 0.0)` becomes equivalent to an `X_ERROR(0.1)` followed by a `Z_ERROR(0.2)`.
  
  This argument can also be set to a probability between 0 and 1, setting a threshold below which the approximation is acceptable. Any error mechanisms that have a component probability above the threshold will cause an exception to be thrown.
  
  <Note>
    This assumption is an approximation, but it is a good approximation for small probabilities.
  </Note>
</ParamField>

<ParamField path="ignore_decomposition_failures" type="bool" default="False">
  When set to true, circuit errors that fail to decompose into graphlike detector error model errors no longer cause the conversion process to abort. Instead, the undecomposed error is inserted into the output.
  
  Whatever tool the detector error model is then given to is responsible for dealing with the undecomposed errors (e.g., a tool may choose to simply ignore them).
  
  <Note>
    This parameter is irrelevant unless `decompose_errors=True`.
  </Note>
</ParamField>

<ParamField path="block_decomposition_from_introducing_remnant_edges" type="bool" default="False">
  Requires that both A B and C D be present elsewhere in the detector error model in order to decompose A B C D into A B ^ C D. Normally, only one of A B or C D needs to appear to allow this decomposition.
  
  Remnant edges can be a useful feature for ensuring decomposition succeeds, but they can also reduce the effective code distance by giving the decoder single edges that actually represent multiple errors in the circuit (resulting in the decoder making misinformed choices when decoding).
  
  <Note>
    This parameter is irrelevant unless `decompose_errors=True`.
  </Note>
</ParamField>

### Return Value

<ResponseField name="return" type="stim.DetectorErrorModel">
  A detector error model describing the error processes in the circuit, with support for non-deterministic observables if enabled.
</ResponseField>

### Example

```python
import stim
from tsim.noise.dem import get_detector_error_model

# Create a simple surface code circuit
circuit = stim.Circuit("""
    # Initialize qubits
    R 0 1 2 3 4
    # Apply gates with noise
    H 0 2 4
    DEPOLARIZE1(0.001) 0 1 2 3 4
    CNOT 0 1 2 1 2 3 4 3
    DEPOLARIZE2(0.001) 0 1 2 1 2 3 4 3
    # Measurements
    M 0 1 2 3 4
    DETECTOR rec[-1] rec[-2]
    DETECTOR rec[-3] rec[-4]
    OBSERVABLE_INCLUDE(0) rec[-5]
""")

# Generate detector error model with non-deterministic observables
dem = get_detector_error_model(
    circuit,
    allow_non_deterministic_observables=True,
    approximate_disjoint_errors=True
)

print(dem)
```

## Key Features

### Non-Deterministic Observable Support

The primary feature of this function is support for non-deterministic observables. This is achieved through a conversion process:

1. Convert logical observables (OBSERVABLE_INCLUDE) into detectors (DETECTOR)
2. Call Stim's detector error model generation with `allow_gauge_detectors=True`
3. Convert the new detectors back into logical observables in the DEM

This process allows the error model to handle cases where observables are not deterministic under noiseless execution, which is common in gauge-fixing scenarios.

### Error Decomposition

When `decompose_errors=True`, the function attempts to break down composite errors into simpler components. This is particularly useful for decoders that work best with graphlike error models.

<Warning>
  Error decomposition is not supported when `allow_non_deterministic_observables=True`. Attempting to use both will raise a `ValueError`.
</Warning>

### Loop Handling

For circuits with repetitive structure (REPEAT blocks), the function can identify periodic steady states and represent them efficiently with repeat blocks in the DEM, unless `flatten_loops=True` is specified.

## Implementation Notes

<Note>
  The function flattens the input circuit and processes all OBSERVABLE_INCLUDE instructions, combining those with the same index and updating measurement record indices accordingly. This ensures correct handling of multiple OBSERVABLE_INCLUDE instructions with the same observable index.
</Note>

<Warning>
  The function validates that the number of observables remains consistent after the conversion process. If Stim interprets observables as gauges and removes them, a `ValueError` is raised with details about the error model.
</Warning>
